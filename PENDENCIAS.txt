Arquivo de Pendências e Melhorias Futuras (Backlog)
1. Segurança e Infraestrutura
[ ] Variáveis de Ambiente: Remover senhas e segredos (JWT Secret, Senha do Banco) do application.properties e usar variáveis do sistema (${JWT_SECRET}) para produção.

[ ] CORS: Configurar o WebConfig para permitir que o Front-end (React/Vue) acesse a API quando estiverem em domínios diferentes.

[ ] Docker de Produção: Criar um Dockerfile otimizado para gerar o .jar final da aplicação, não apenas o ambiente de desenvolvimento.

[ ] Testes Automatizados: Implementar testes unitários (JUnit) para as regras de negócio nos Services (UsuarioService, MateriaService, etc).

2. Módulo de Identidade (Usuário)
[ ] Recuperação de Senha: Implementar fluxo de "Esqueci minha senha" com envio de e-mail (Token temporário).

[ ] Confirmação de E-mail: Bloquear o login até que o usuário clique no link de confirmação enviado por e-mail.

[ ] Perfis de Acesso (Roles): Implementar a lógica de ADMIN vs USER (atualmente está fixo como ROLE_USER).

[ ] Logout: Implementar "Blacklist" de tokens ou tempo de expiração curto com Refresh Token para invalidar sessões.

3. Módulo de Inventário (Matérias e Tópicos)
[ ] Integridade na Exclusão: Antes de excluir uma Matéria, verificar se ela já possui Registros de Estudo ou vínculo com Concursos. Se tiver, impedir a exclusão ou arquivar (Soft Delete).

[ ] Importação em Lote: Permitir importar matérias via arquivo (CSV/Excel).

4. Módulo de Estratégia (Concursos)
[ ] Status do Concurso: Criar campo ou lógica para definir se o concurso está "Aberto", "Edital Publicado" ou "Encerrado".

[ ] Dashboard: Exibir contagem regressiva para a prova (dias restantes).

5. Módulo de Planejamento (Ciclos) - O próximo grande passo
[ ] Vínculo Concurso-Matéria: Tabela intermediária para definir Pesos e Metas de horas.

[ ] Algoritmo do Ciclo: Lógica matemática para distribuir as horas disponíveis baseado nos pesos.

[ ] Histórico de Ciclos: Permitir arquivar um ciclo e iniciar outro sem perder os dados anteriores.

[ ] Mostrar qual matéria estudar agora: Calcular e ver qual matéria está mais longe para fechar o ciclo e recomendar o estudo dela.

[ ] Visualização da Estratégia: Endpoint para listar todas as matérias de um concurso mostrando os dados (GET /concursos/{id}/materias).

[ ] Edição do Vínculo: Permitir alterar os dados (peso quantidade de questões) de uma matéria já vinculada.

[ ] Desvincular: Permitir remover uma matéria do concurso (sem apagar a matéria do banco).

[ ] Importação de Edital: Futuramente, permitir ler um PDF ou colar um texto para preencher automaticamente as matérias e quantidade de questões.

[ ] Algoritmo de Metas de Questões: Criar lógica que define uma "Meta de Questões a Resolver" baseada na "Quantidade de Questões da Prova" x "Peso" x "Tempo até a prova".

[ ] Simulador de Nota: Com base nos pesos cadastrados, criar uma calculadora onde o usuário coloca quantos acertos espera ter e o sistema calcula a nota final provável.

[ ] Edição de Vínculo: Permitir alterar o Peso ou a Qtd. de Questões de uma matéria já vinculada sem ter que remover e adicionar de novo.

[ ] Edição de Ciclo Fechado: (Decisão de Produto) Se o usuário insistir em editar um ciclo fechado, teremos que recalcular todos os saldos posteriores. Por enquanto, bloquear edição de ciclos fechados para manter a integridade.

[ ] Dashboard Geral: Criar uma visão que mostre "Horas Totais de Vida" vs "Horas em Concursos Ativos".

[ ] Migração de Estudo: Permitir que o usuário pegue um estudo "Sem Vínculo" e atribua a um concurso posteriormente (caso tenha esquecido).
